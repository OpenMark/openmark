OpenMark coding guidelines

General rules

  • Indent control structures with a single tab character.
  • Indent code wrapped to the next line with two tabs.
  • No trailing whitespace.
  • No Hungarian notation. That is, use variable names like String title, not String sTitle
  • Put the braces on a separate line (see the templates below).
  • No space around arithmetic operators or in control statements.
  • Single space around logical operators.
