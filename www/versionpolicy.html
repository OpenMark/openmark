OpenMark version number policy

We use a linux-like policy.

  • Stable releases are have numbers like 1.2.0, 1.4.0, 1.6.0, and so on.
  • Minor bug and security fixes to the 1.4.0 release will be numbered 1.4.1, 1.4.2, etc. and developed off a stable branch in subversion.
  • After the 1.4.0 release, new development will take place of a 1.5.x, series of development snap-shots.

On the status page of a running OpenMark server, the version number displayed will look like

Om version: 1.4.0 (2007-05-09 18:15:11)

Where the timestamp in brackets was the time the system was compiled.

The version number of the codebase is stored in version.properties file.
